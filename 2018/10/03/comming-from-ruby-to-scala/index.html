从 Ruby 到 Scala

发表于 2018-10-03 | 更新于 2018-10-04 | 分类于 程序设计语言

变量

程序设计语言是计算机系统的一个抽象分层，程序设计语言的实现可以看作是程序设计语言到计算机指令集体系结构的一个映射。
变量是程序设计语言中的一个重要概念，并不存在于计算机指令集体系结构中，可以视为对存储操作的抽象，可以通过硬件或软件支持。
我们可以使用名字、地址、数值、类型、生存期、作用域来刻画一个变量，理解绑定有助于我们理解变量的相关概念。

一般地，绑定 是一种关联，如存在于属性与实体之间的关联，或者操作与符号之间的关联。

绑定可以发生在语言设计时、语言实现时、编译时、载入时、连接时或者运行时。

绑定发生的时间称为 绑定时间。

彻底地理解程序实体属性的绑定时间是理解程序设计语言语义的前提。

类型

类型决定了变量的取值范围及关联的一组操作集合。

Scala 采用静态类型；Ruby采用动态类型。Scala 需要借助 类型推理、类型转换、类型参数 来实现Ruby的灵活性。我们不得不去了解类型检测、类型错误、类型兼容等概念。

常量

命名常量 是只与值绑定一次的变量。

Ruby 常量是可以多次绑定值的，虽然Ruby 解释器会给出警告。

惊讶

Scala 由类型检测、类型转换、类型错误、类型兼容、类型推理、类型参数构成的类型相关概念需要花时间学习。

数据类型

数据类型 定义一组数据值，以及在这些数值上的预定义的一组操作。

数据 -> 数据类型 -> 自定义数据类型 -> 抽象数据类型 -> 面向对象的程序设计。

基本数据类型 是指不用其他类型来定义的数据类型，一般包括数值类型（整数、浮点数、负数、小数）、布尔类型、字符类型、字符串类型。

高级数据类型 包括枚举类型、子范围类型、数组类型（异构数组/关联数组）、记录类型、联合类型、指针类型、引用类型

数组

映射

映射 是 键/值 对偶的集合。

元组

映射是键/值对偶的集合。对偶是元组的最简单形态——元组是不同类型的值的聚集。

集合

Seq 是一个有先后次序的值的序列，如数组或列表。IndexedSeq 允许通过整型的下标快速地访问任意元素。

Set 是一组没有先后次序的值。在 SortedSet 中，元素以某种排过序的顺序被访问。

Map 是一组键/值对偶。SortedMap 按照键的排序访问其中的实体。

每个Scala集合特质或类都有一个带有apply方法的伴生对象，这个apply方法可以用来构建该集合中的实例。这样的设计叫做 统一创建原则。

Scala同时支持可变的和不可变的集合。Scala优先采用不可变集合，可以基于老的集合创建新的集合。

序列

Vector 是 ArrayBuffer 的不可变版本：一个带下标的序列，支持快速的随机访问。

Range 表示一个整数序列。Range 对象并不存储所有值而只是起始值、结束值和增值。可以使用 to/until 构造 Range 对象。

栈、队列、优先级队列等都是标准的数据结构，用来实现特定算法。

列表

在 Scala 中，列表要么是Nil（空表），要么是一个head元素加上一个tail，而tail又是一个列表。

可变列表

可变的LinkedList和不可变的List相似，只不过可以通过对elem引用赋值来修改其头部，对next引用赋值来修改其尾部。

除了 LinkedList外，Scala还提供了一个DoubleLinkedList，区别是多带一个prev引用。

集

集是不重复元素的集合。尝试将已有元素加入没有效果。

和列表不同，集并不保留元素插入的顺序。缺省情况下，集是以哈希集实现的，其元素根据hashCode方法的值进行组织。

流

流提供的是一个不可变的替代品。流是一个尾部被懒计算的不可变列表——也就是说，只有当你需要时它才会被计算。

惊讶

  • Ruby 不区分字符和字符串；Scala 支持 Char 和 String 类型
  • Ruby 不支持对偶、元组，集合类型没有 Scala 细化
  • Scala 支持更多的数据结构，尤其是对流和懒视图的支持

表达式与赋值语句

在 Scala 中，每个表达式都有一个类型。

Scala 区分表达式和语句。

标识符

变量、函数、类等的程序实体的名称统称为标识符。

结合性

在 Scala 中，所有操作符都是左结合，除了：

  • 以冒号结尾的操作符
  • 赋值操作符

return 表达式

语句层次的控制结构

事实已经证明，所有能够由流程图表述的算法都能够使用两种控制语句在程序设计语言中编码：一种是在两条控制流程路径之间的选择语句，第二种是逻辑控制的循环语句。无条件转移语句是多余的。

选择语句

选择语句 在程序中提供在两个或多个执行路径之间进行选择的方法。选择语句可以分为双向选择和多向选择。

Ruby 支持if语句、unless语句、case语句；Scala 不支持 unless 语句、case 语句，Scala 支持模式匹配。

Scala 的模式匹配很强大。

循环语句

语句的重复执行，在一种函数式语言中通常是由递归而不是循环构造来完成的。

Ruby 支持 while 语句、for 语句、until 语句、times 方法、each 方法、loop 方法；Scala 支持 while 语句、for 语句(for 推导式)。

Ruby 支持 break、next、redo；Scala 不支持，控制权的转移是通过抛出和捕获异常完成的。

子程序

Scala 支持函数、过程、方法。

子程序实现

Ruby 调用方法需要两步：第一步是查找方法，第二步是执行方法。

抽象数据类型和封装结构

在程序设计语言的世界里，抽象是对抗程序设计复杂性的一种武器，其目的是简化程序设计的过程。
抽象允许程序人员将注意力集中于基本属性而忽略次要属性。
当代程序设计语言中的两类基本抽象是过程抽象和数据抽象。

类型参数

在 Scala 中，可以用类型参数来实现类和函数，这样的类和函数可以用于多种类型。

在 Scala 中，可以指定类型如何根据类型参数的变化而变化。

隐式转换

所谓 隐式转换函数（implicit conversation function) 指的是以 implicit 关键字声明的带有单个参数的函数。隐式转换函数将被自动应用，将值从一种类型转换为另一种类型。

Scala 会考虑如下的隐式转换函数：

  1. 位于源或目标类型的伴生对象中的隐式函数
  2. 位于当前作用域可以以单个标识符指代的隐式函数

隐式转换在如下三种不同的情况会被考虑：

  1. 当表达式的类型与预期的类型不同时
  2. 当对象访问一个不存在的成员时
  3. 当对象调用某个方法，而该方法的参数声明与传入参数不匹配时

有三种情况编译器不会尝试使用隐式转换：

  1. 如果代码能够在不使用隐式转换的前提下通过编译，则不会使用隐式转换
  2. 编译器不会尝试同时执行多个转换
  3. 存在二义性的转换是个错误

隐式参数

函数或方法可以带有一个标记为 implicit 的参数列表。这种情况下，编译器将会查找缺省值，提供给该函数或方法。

面向对象的程序设计

面向对象的程序设计，是抽象数据类型的抽象原理的一种应用。

一种面向对象语言必须支持三个关键的语言特性：封装、继承、多态。

抽象类

在 Scala 中，用 abstract 关键字来标记不能被实例化的类，抽象类包含某个或某几个没有被完整定义的方法。
抽象方法 就是一个没有方法体的方法
抽象字段 就是一个没有初始值的字段

Ruby 不支持抽象类。

特质

在 Scala 中，特质中的方法并不需要一定是抽象的，特质中的字段可以是具体的，也可以是抽象的。
在 Scala 中，缺少构造器参数是特质与类之间唯一的技术差别。

继承

几乎在所有的情况下，抽象数据类型复用中的问题，都是已有类型的特性和功能不适合于新的使用。

继承提供了一种解决抽象数据类型的复用所面临的修改问题，以及程序组织问题的办法。

继承是作为增加复用可能性的一种手段，然而它的缺点则是产生了在继承层次结构中的类之间的相互依赖性。

扩展类

继承层级

与 Java 中基本类型对应的类，以及 Unit 类型，都扩展自 AnyVal；所有其他类都是 AnyRef 的子类，AnyRef 是 Java 虚拟机中 Object 类的同义词。

AnyVal 和 AnyRef 都扩展自 Any 类，而 Any 类是整个继承层级的根节点。

Any 类定义了 isInstanceOf/asInstanceOf 方法，以及用于相等性判断和哈希码的方法。

AnyVal 并没有追加任何方法。它只是所有值类型的一个标记。

AnyRef 类追加了来自 Ojbect 类的监视方法 wait/nofifyAll。同时提供了一个带函数参数的方法 synchronized。

所有 Scala 类都实现了 ScalaObject 这个标记接口，这个接口没有定义任何方法。

在继承层级的另一端是 Nothing 和 Null 类型。

Null 类型的唯一实例是 null 值。可以将 null 赋值给任何引用，但不能赋值给值类型的变量。

Nothing 类型没有实例。它对于泛型结构时常有用。

闭包

在 Scala 中，可以在任何作用域内定义函数：包、类甚至是另一个函数或方法。

在函数内，可以访问到相应作用域内的任何变量。需要注意，函数可以在变量不再处于作用域内时被调用。

闭包由代码和代码用到的任何非局部变量定义构成。

柯里化

柯里化指的是将原来接受两个参数的函数变成新的接受一个参数的函数的过程。新的函数返回一个以原有第二个参数作为参数的函数。
